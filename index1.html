Integration with Cloudflare Tunnel

Examples of how to integrate these proxy sources with Cloudflare Tunnel for secure access to your applications.

Using cloudflared with Proxy Configuration 
# Example configuration for using proxies with Cloudflare Tunnel
# In your cloudflared config file (~/.cloudflared/config.yml)

tunnel: "your-tunnel-id"
credentials-file: "/path/to/credentials.json"

ingress:
  - hostname: "app.yourdomain.com"
    service: "http://localhost:3000"
    originRequest:
      proxyType: "socks"
      proxyAddress: "127.0.0.1:1080"
  - service: "http_status:404"

# For HTTP proxies, use:
# proxyType: "http"
# proxyAddress: "proxy.example.com:8080"
Bash Script to Fetch and Rotate Proxies 
#!/bin/bash
# Script to fetch proxies and configure cloudflared

fetch_proxies() {
    # Fetch from your GitHub proxy list
    curl -s https://raw.githubusercontent.com/AFRcloud/ProxyList/main/ProxyList2.txt > proxies.txt
    
    # Extract Malaysian proxies (MY) as an example
    grep " MY$" proxies.txt > my_proxies.txt
    echo "Fetched $(wc -l < my_proxies.txt) Malaysian proxies"
}

setup_proxy_rotation() {
    # Simple round-robin proxy rotation
    while true; do
        for proxy in $(cat my_proxies.txt); do
            ip=$(echo "$proxy" | cut -d: -f1)
            port=$(echo "$proxy" | cut -d: -f2)
            
            # Update cloudflared config with current proxy
            sed -i "s/proxyAddress: \".*\"/proxyAddress: \"$ip:$port\"/" ~/.cloudflared/config.yml
            
            # Restart cloudflared to apply changes
            pkill cloudflared
            cloudflared tunnel run your-tunnel-id &
            
            # Use this proxy for 5 minutes
            sleep 300
        done
    done
}

# Main execution
fetch_proxies
setup_proxy_rotation
Docker Compose with Proxy Support 
# docker-compose.yml
version: '3.8'

services:
  cloudflared:
    image: cloudflare/cloudflared:latest
    command: tunnel run
    environment:
      - TUNNEL_URL=http://webapp:8080
      # Proxy configuration via environment variables
      - HTTP_PROXY=http://proxy-server:8080
      - HTTPS_PROXY=http://proxy-server:8080
    volumes:
      - ./config:/etc/cloudflared
    depends_on:
      - webapp
      - proxy-server

  webapp:
    image: your-web-app:latest
    ports:
      - "8080:8080"

  proxy-server:
    image: squidproxy/squid:latest
    ports:
      - "3128:3128"
    volumes:
      - ./squid.conf:/etc/squid/squid.conf"
